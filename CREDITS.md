# Credits & Art Licensing

StrongBow ships with **100% original, procedurally-generated art** (see `src/rendering/`).
You can override any of it with external image files through the asset pipeline
described below — but **only with art you have the right to use**.

## External art pipeline

- Overrides live in **`public/assets/manifest.json`**.
- Each entry replaces the built-in procedural texture for a given `key` with a PNG.
- Anything missing or failing to load **falls back to procedural art**, so the
  game always runs.
- `manifest.json` is read at runtime, so you can add/remove art and just refresh.

To revert any override, delete its entry from `manifest.json` (or delete the PNG).

### Special keys

| Key | What it replaces |
| --- | --- |
| `ext-floor` | the dungeon floor tile (retiles the whole map) |
| `ext-wall` | the dungeon wall tile |
| `chest`, `chest-open`, `key`, `gem`, `crystal`, `cog`, `skull-pike`, `banner`, `bones`, `rubble` | loot & decor |
| `potion-red`, `potion-blue`, `food` | pickups |
| `hero-vanguard-sheet`, `hero-thief-sheet`, `hero-arcanist-sheet`, `hero-warden-sheet` | hero sprite sheets (`kind: "spritesheet"`) |
| `monster-grunt-sheet`, `monster-ghost-sheet`, `monster-demon-sheet`, `monster-bone_archer-sheet`, `monster-brute-sheet`, `monster-imp-sheet`, `monster-boss-sheet`, `monster-molten_colossus-sheet` | enemy sprite sheets |

## Bundled demo art

All files in `public/assets/sprites/` are **original work by the StrongBow
project, released under [CC0 1.0](https://creativecommons.org/publicdomain/zero/1.0/)
(public domain)** — use, modify, or delete freely:

- Tiles & decor (`floor`, `wall`, `chest`, `chest-open`, `crystal`, `cog`,
  `skull-pike`) — generated by `tools/gen_demo_tiles.py`.
- **Hero sheets** (`warrior`, `elf`, `wizard`, `valkyrie`) — original
  top-down arcade designs generated by `tools/gen_heroes.py`. 240×24, 12 frames
  (3 facings × idle/walk/walk/attack). Mapped onto Vanguard / Thief / Arcanist
  / Warden in `manifest.json`. These are inspired-by, not copied-from, any
  existing game.

## Using CC0 packs (e.g. Kenney)

[Kenney](https://kenney.nl) and [OpenGameArt](https://opengameart.org) publish
excellent **CC0 (public-domain)** top-down dungeon art. Recommended:

- **Tiny Dungeon** — https://opengameart.org/content/tiny-dungeon (`kenney_tinydungeon.zip`, CC0)
- **Roguelike/RPG pack** — https://kenney.nl/assets/roguelike-rpg-pack (CC0)
- **Micro Roguelike** — https://kenney-assets.itch.io/micro-roguelike (CC0)

Drop-in steps:
1. Download and unzip the pack.
2. Copy the individual tile PNGs you want into `public/assets/sprites/`.
3. Add entries to `public/assets/manifest.json` mapping each file to a StrongBow `key`.
4. Refresh the game.

Crediting Kenney is appreciated but not legally required for CC0.

## Dropping in your own hero sprites

The four classes map cleanly onto the classic archetypes:

| StrongBow class | Texture key | Archetype |
| --- | --- | --- |
| Vanguard | `hero-vanguard-sheet` | Warrior |
| Thief | `hero-thief-sheet` | Hooded rogue / dagger |
| Arcanist | `hero-arcanist-sheet` | Wizard |
| Warden | `hero-warden-sheet` | Valkyrie |

The built-in hero sheet is a **12-frame grid, 4 columns × 3 rows**, frame size
20×24 px, in this order:

```
row 0 (facing DOWN):  idle  walkA  walkB  attack
row 1 (facing UP):    idle  walkA  walkB  attack
row 2 (facing SIDE):  idle  walkA  walkB  attack     (left = side flipped)
```

To use your own, put the PNG in `public/assets/sprites/` and add a manifest entry:

```json
{ "key": "hero-vanguard-sheet", "path": "assets/sprites/warrior.png",
  "kind": "spritesheet", "frameWidth": 20, "frameHeight": 24 }
```

If your sheet has a **different size or frame order** (most downloaded sheets do),
that's fine — drop the files in `public/assets/sprites/` and I can write a small
adapter that re-slices them onto this layout. Tell me each sheet's pixel size and
how its frames are arranged (or just let me inspect the files).

## ⚠️ A note on ripped game sprites

Sprites extracted from commercial games (e.g. Gauntlet / Gauntlet II on sprite-rip
sites) remain **copyrighted by their rights holders**, even for a personal,
non-commercial project. "For personal use" labels on those sites are not a license
to ship the art in your own game. This project only bundles original CC0 art, and
the pipeline above is intended for art you own or that is properly licensed (CC0 /
public domain / your own work). This is not legal advice.
